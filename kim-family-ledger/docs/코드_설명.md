# Kim Family 가계부 코드 가이드

이 문서는 나중에 새 기능을 만들 때 지켜야 할 기본 구조와 UI 톤을 설명한다.  
목표는 "부모님도 헷갈리지 않고 쓸 수 있는, 오프라인 가족 가계부"다.

## 앱 기본 정보

| 항목 | 값 |
| --- | --- |
| applicationId / namespace | `com.kimfamily.ledger` |
| 런처 이름 | `가계부` |
| 앱 내부 기본 이름 | `Kim Family 가계부` |
| 저장 방식 | Room SQLite, 완전 오프라인 |
| 주요 스택 | Kotlin, Jetpack Compose, Material3, Room, Navigation Compose, StateFlow |

런처 이름은 `@string/launcher_name`, 앱 내부 기본 이름은 `@string/app_name`을 사용한다.  
설정에서 바꾸는 앱 이름은 런처 이름이 아니라 앱 내부 헤더, 시작 화면, 앱 정보, 공유 제목에 적용된다.

## 구조

```text
UI Screen
  -> ViewModel
  -> LedgerRepository
  -> Room DAO
  -> Entity

Domain Model은 UI와 DB 사이의 공통 언어로 사용한다.
```

주요 패키지:

```text
com.kimfamily.ledger
├── LedgerApplication.kt
├── MainActivity.kt
├── navigation/
├── domain/model/
├── data/
│   ├── AppPreferences.kt
│   ├── local/
│   ├── mapper/
│   └── repository/
└── ui/
    ├── home/
    ├── transaction/
    ├── stats/
    ├── settings/
    ├── components/
    ├── theme/
    └── util/
```

소스 파일 경로가 아직 `app/src/main/java/com/kyungah/ledger` 아래에 있어도 Kotlin package는 `com.kimfamily.ledger`다. 새 파일을 만들 때는 기존 폴더에 추가해도 빌드는 되지만, 장기적으로는 폴더 경로도 package에 맞춰 정리하는 것이 좋다.

## 데이터 모델

핵심 모델:

- `TransactionType`: `EXPENSE`, `INCOME`
- `Transaction`: 금액, 타입, 카테고리, 메모, 발생 시각
- `Category`: 타입별 카테고리, 아이콘, 색상, 기본 여부
- `MonthlySummary`: 수입 합계, 지출 합계, `netTotal`
- `CategorySummary`: 카테고리별 합계

저장 규칙:

- 금액은 원 단위 `Long`으로 저장한다.
- 날짜/시간은 epoch millis로 저장한다.
- 지출과 수입 카테고리는 분리한다.
- 기본 카테고리는 삭제할 수 없다.
- 거래에 사용 중인 카테고리는 삭제할 수 없다.

## Repository 기준

앱의 데이터 접근은 `LedgerRepository`를 거친다.

주요 API:

- `observeTransactions(year, month, query)`
- `observeMonthlySummary(year, month)`
- `observeCategorySummaries(year, month, type)`
- `saveTransaction(transaction)`
- `deleteTransaction(id)`
- `saveCategory(category)`
- `deleteCategory(category)`
- `exportCsv(year, month)`  
  UI에서는 "텍스트로 내보내기"라고 표현한다.
- `exportBackupJson()`
- `importBackupJson(json)`

새 기능을 만들 때 UI에서 DAO를 직접 호출하지 않는다. 필요한 쿼리는 DAO에 추가하고 Repository 메서드로 감싼다.

## 화면 구성

### 홈

파일:

- `HomeScreen.kt`
- `HomeViewModel.kt`
- `TransactionListItem.kt`
- `SummaryCard.kt`
- `MonthSelector.kt`

원칙:

- 월 선택 영역은 좁은 화면에서도 `이번 달` 배지가 월 텍스트와 겹치면 안 된다.
- 목록은 날짜 그룹 헤더와 항목 시간을 분리한다.
  - 헤더: `오늘 (토)`, `5월 16일 (토)`
  - 항목: `14:30`
- 날짜가 이미 헤더에 있으므로 항목 안에 날짜를 반복하지 않는다.
- 검색창 입력은 로컬 상태로 받고, 검색 반영은 약간 지연시켜 한글 조합 입력이 깨지지 않게 한다.
- 추가 버튼은 리스트를 가리지 않는다. 현재는 상단 액션 버튼을 사용한다.

### 거래 추가/수정

파일:

- `TransactionFormScreen.kt`
- `TransactionFormViewModel.kt`

원칙:

- 수입/지출은 수정 화면에서도 바꿀 수 있다.
- 타입을 바꾸면 카테고리 목록을 새 타입에 맞게 다시 불러오고 선택을 초기화한다.
- 날짜/시간은 바텀시트 UX를 사용한다.
  - 본문에는 선택값 카드만 보여준다.
  - 바텀시트에는 `오늘`, `어제`, `지금` 빠른 선택을 둔다.
  - 필요할 때만 날짜 선택, 시간 선택을 연다.
- 빈 영역 탭 시 포커스를 해제해서 키보드가 내려가야 한다.
- 금액/메모 입력창은 흰색 surface 배경과 중립 테두리를 사용한다.

### 통계

파일:

- `StatsScreen.kt`
- `StatsViewModel.kt`
- `CategoryBreakdownChart.kt`

원칙:

- 요약 카드는 홈과 동일한 `SummaryCard`를 사용한다.
- `순수지`라는 표현은 쓰지 않는다. 현재 UI 라벨은 `차액`이다.
- 카테고리별 통계에는 홈 리스트와 같은 카테고리 아이콘을 표시한다.
- 수입/지출 탭에 따라 제목은 `카테고리별 수입`, `카테고리별 지출`로 표시한다.

### 설정

파일:

- `SettingsScreen.kt`
- `CategoryListScreen.kt`
- `CategoryListViewModel.kt`

설정은 섹션형 UI를 사용한다.

- 카테고리
- 데이터
- 앱 정보

기능:

- 수입/지출 카테고리 관리
- 이번 달 텍스트로 내보내기
- 백업 파일 만들기
- 백업 가져오기
- 앱 내부 이름 바꾸기

앱 이름 수정은 `AppPreferences`에 저장한다. 런처 이름은 Android Manifest 리소스라 앱 안에서 즉시 바꾸지 않는다.

## UI 톤앤매너

키워드:

- 따뜻함
- 조용함
- 부모님도 읽기 쉬움
- 장부처럼 명확함
- 과한 장식 없음

색상:

- 전체 배경은 흰색/중립 베이지 계열을 기본으로 한다.
- 테라코타 계열은 포인트 색으로만 사용한다.
- 지출은 부드러운 빨강 계열.
- 수입은 초록을 중심으로 하되 카테고리 구분을 위해 보라, 청록, 브라운 계열을 일부 섞는다.
- 지나치게 형형색색이 되지 않게 한다. 카테고리 구분은 필요하지만 수입/지출의 큰 의미 구분이 우선이다.

컴포넌트:

- 리스트 항목은 카드가 둥둥 떠 보이기보다 흰색 행 + 얇은 구분선 느낌을 유지한다.
- 반복 리스트는 아이콘, 이름, 보조 텍스트, 금액의 위치를 일관되게 둔다.
- 좁은 화면에서 텍스트가 겹치면 안 된다. `maxLines`, `overflow`, 적절한 `weight`를 반드시 고려한다.
- 앱 내 문구는 너무 개발자스럽거나 회계 문서처럼 딱딱하면 안 된다.

표현 가이드:

| 피할 표현 | 권장 표현 |
| --- | --- |
| CSV 보내기 | 텍스트로 내보내기 |
| 순수지 | 차액 |
| 이 달 | 이번 달 |
| 내역이 없습니다 | 표시할 내역이 없어요 |
| 벌어들인 돈 | 들어온 돈 |

## 백업과 복원

자동 클라우드 동기화는 없다. 데이터는 기본적으로 기기 내부 Room DB에 저장된다.

설정에서 제공하는 백업:

- `백업 파일 만들기`: 카테고리와 거래 전체를 JSON으로 저장
- `백업 가져오기`: JSON 백업 파일로 현재 데이터를 교체

복원은 기존 데이터를 백업 파일 내용으로 교체한다. UX에서는 반드시 확인 다이얼로그를 보여준다.

## 새 코드 작성 체크리스트

- 화면 문구가 부모님이 읽기 쉬운가?
- 작은 화면에서 텍스트가 겹치지 않는가?
- 거래 리스트를 가리는 플로팅 UI가 없는가?
- 날짜와 시간이 중복 표시되지 않는가?
- 새 데이터 접근은 Repository를 통하는가?
- 수입/지출 타입 변경 시 카테고리 정합성이 유지되는가?
- 백업/복원 시 카테고리와 거래 FK가 깨지지 않는가?
- `./gradlew :app:compileDebugKotlin`이 성공하는가?
