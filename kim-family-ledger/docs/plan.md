# Kim Family 가계부 요구사항

이 문서는 앱이 제공해야 하는 기능과 동작 기준을 정리한다.  
개발 구조와 코드 작성 기준은 [`코드_설명.md`](코드_설명.md)를 참고한다.

## 1. 제품 목표

부모님이 매일 수입과 지출을 쉽게 기록하고, 이번 달 흐름을 한눈에 확인할 수 있는 오프라인 가계부 앱을 만든다.

핵심 원칙:

- 빠르게 기록할 수 있어야 한다.
- 화면 문구와 흐름이 쉬워야 한다.
- 작은 화면에서도 정보가 겹치지 않아야 한다.
- 데이터는 기기 내부에 저장하고, 필요하면 백업 파일로 옮길 수 있어야 한다.

## 2. 앱 기본 정보

| 항목 | 요구사항 |
| --- | --- |
| applicationId | `com.kimfamily.ledger` |
| 런처 이름 | `가계부` |
| 앱 내부 기본 이름 | `Kim Family 가계부` |
| 앱 내부 이름 수정 | 설정에서 가능 |
| 동작 방식 | 오프라인 전용 |
| 데이터 저장 | Room DB |
| 백업 방식 | JSON 파일 내보내기/가져오기 |

앱 내부 이름 수정은 헤더, 시작 화면, 설정 앱 정보, 공유 제목에 반영한다.  
런처 이름은 `가계부`로 고정한다.

## 3. 주요 사용자

- 부모님 또는 가족 구성원
- 복잡한 금융 앱보다 단순한 수입/지출 기록이 필요한 사용자
- 클라우드 계정이나 로그인 없이 로컬에서 쓰고 싶은 사용자

## 4. 홈 요구사항

### 4.1 월 선택

- 이전 달, 다음 달 이동이 가능해야 한다.
- 현재 달이면 `이번 달` 배지를 보여준다.
- 작은 화면에서도 월 텍스트와 `이번 달` 배지가 겹치면 안 된다.

### 4.2 월 요약

- 수입 합계, 지출 합계, 차액을 보여준다.
- `순수지`라는 표현은 사용하지 않는다.
- 금액이 길어도 칸 밖으로 넘치거나 겹치면 안 된다.

### 4.3 거래 목록

- 월별 거래를 날짜 그룹으로 보여준다.
- 날짜 그룹 헤더에는 요일을 포함한다.
  - 예: `오늘 (토)`, `어제 (금)`, `5월 16일 (토)`
- 각 거래 항목에는 다음 정보를 보여준다.
  - 카테고리 아이콘
  - 카테고리 이름
  - 메모
  - 금액
  - 시간
- 날짜는 그룹 헤더에 있으므로 거래 항목에는 시간만 보여준다.
- 리스트 항목 사이에는 얇은 구분선을 사용한다.
- 추가 버튼이나 다른 UI가 리스트를 가리면 안 된다.

### 4.4 검색

- 메모 검색을 지원한다.
- 한글 조합 입력이 밀리거나 깨지지 않아야 한다.
- 검색창을 닫으면 검색어도 초기화되어 숨은 필터가 남지 않아야 한다.

## 5. 거래 추가/수정 요구사항

### 5.1 타입

- 거래 타입은 수입과 지출이다.
- 신규 거래와 기존 거래 모두 타입 변경이 가능해야 한다.
- 타입을 바꾸면 카테고리 목록은 해당 타입에 맞게 바뀌고, 기존 선택 카테고리는 초기화된다.

### 5.2 금액

- 숫자만 입력할 수 있어야 한다.
- 원 단위 정수로 저장한다.
- 입력 중 포맷된 금액을 보조 텍스트로 보여준다.

### 5.3 카테고리

- 선택한 타입에 해당하는 카테고리만 보여준다.
- 카테고리는 칩 형태로 선택한다.
- 카테고리를 선택하지 않으면 저장할 수 없다.

### 5.4 날짜와 시간

- 본문에는 선택된 날짜/시간을 카드 형태로 보여준다.
- 누르면 바텀시트가 열린다.
- 바텀시트에는 빠른 선택을 제공한다.
  - 오늘
  - 어제
  - 지금
- 직접 선택이 필요하면 날짜 선택, 시간 선택을 별도로 열 수 있어야 한다.
- 날짜와 시간 선택은 사용 흐름을 과하게 끊지 않아야 한다.

### 5.5 메모

- 메모는 선택 입력이다.
- 입력창 배경은 흰색 surface 계열을 사용한다.

### 5.6 키보드

- 입력 중 빈 영역을 탭하면 키보드가 내려가야 한다.
- Compose 구현 기준으로는 포커스를 해제한다.

## 6. 통계 요구사항

- 월별 통계를 볼 수 있어야 한다.
- 수입/지출 탭 전환이 가능해야 한다.
- 선택된 타입에 따라 제목을 다르게 보여준다.
  - `카테고리별 지출`
  - `카테고리별 수입`
- 카테고리별 통계에는 카테고리 아이콘을 표시한다.
- 각 항목에는 카테고리 이름, 비율, 금액, 막대 그래프를 보여준다.
- 내역이 없으면 `표시할 내역이 없어요`라고 보여준다.

## 7. 설정 요구사항

설정 화면은 다음 섹션으로 구성한다.

- 카테고리
- 데이터
- 앱 정보

### 7.1 카테고리 관리

- 지출 카테고리와 수입 카테고리를 각각 관리한다.
- 카테고리 목록에는 실제 거래 리스트와 동일한 아이콘을 보여준다.
- 카테고리 추가/수정 시 아이콘을 선택할 수 있어야 한다.
- 기본 카테고리는 삭제할 수 없다.
- 거래에 사용 중인 카테고리는 삭제할 수 없다.

### 7.2 텍스트로 내보내기

- 이번 달 거래 내역을 텍스트로 공유할 수 있어야 한다.
- UI 문구는 `CSV`가 아니라 `텍스트로 내보내기`를 사용한다.
- 공유 제목에는 현재 앱 내부 이름을 포함한다.

### 7.3 백업 파일 만들기

- 전체 카테고리와 전체 거래를 JSON 파일로 저장할 수 있어야 한다.
- 기본 파일명은 `kim-family-ledger-backup.json`이다.

### 7.4 백업 가져오기

- JSON 백업 파일을 선택해 복원할 수 있어야 한다.
- 가져오기 전 확인 다이얼로그를 보여준다.
- 복원하면 현재 데이터는 백업 파일 내용으로 교체된다.

### 7.5 앱 이름 수정

- 사용자는 설정에서 앱 내부 이름을 바꿀 수 있어야 한다.
- 변경된 이름은 다음 위치에 반영한다.
  - 홈 헤더
  - 시작 화면
  - 설정 앱 정보
  - 텍스트 내보내기 공유 제목
- 런처 이름은 `가계부`로 유지한다.

## 8. 기본 카테고리 요구사항

지출 기본 카테고리:

- 식비
- 교통
- 쇼핑
- 주거/공과금
- 문화/여가
- 의료
- 교육
- 기타

수입 기본 카테고리:

- 급여
- 부수입
- 용돈
- 이자/배당
- 기타

지출은 카테고리별 색 구분이 비교적 뚜렷해도 된다.  
수입은 초록 계열을 중심으로 하되 카테고리 구분이 가능하도록 색을 조금 나눈다.

## 9. 데이터와 백업 요구사항

- 앱 설치 직후 거래 데이터는 비어 있어야 한다.
- 데모 거래나 랜덤 거래를 자동 추가하지 않는다.
- 기본 카테고리는 자동으로 준비한다.
- Android 자동 백업에만 의존하지 않는다.
- 사용자가 직접 백업 파일을 만들고 가져올 수 있어야 한다.

## 10. 비기능 요구사항

- 네트워크 권한 없이 동작해야 한다.
- 로그인이나 서버 동기화는 제공하지 않는다.
- 작은 화면에서도 텍스트 겹침이 없어야 한다.
- 부모님이 읽기 쉬운 표현을 사용한다.
- 중요한 데이터 변경, 특히 백업 복원은 확인 절차를 둔다.

## 11. 현재 제외된 기능

- 클라우드 동기화
- 로그인
- 예산/한도 관리
- 반복 거래
- 알림
- 홈 위젯
- 백업 파일 병합 가져오기

## 12. 배포 전 확인

- 새 설치 시 거래가 비어 있는지 확인한다.
- 기본 카테고리가 정상 생성되는지 확인한다.
- 거래 추가, 수정, 삭제가 되는지 확인한다.
- 수입/지출 타입 변경 시 카테고리가 새로 로드되는지 확인한다.
- 백업 파일 만들기와 가져오기가 되는지 확인한다.
- 앱 이름 변경 후 내부 표시가 바뀌는지 확인한다.
- 작은 화면에서 `이번 달` 배지와 월 텍스트가 겹치지 않는지 확인한다.
- 빌드가 성공해야 한다.

```bash
./gradlew :app:compileDebugKotlin
```
